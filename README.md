# 🐾 Cool Cat Kit

**Личный дневник и кабинет кота.** Хозяин ведёт паспорт питомца, следит за
прививками и здоровьем, планирует события в календаре, отслеживает вес по графику,
считает норму корма и собирает «историю» жизни кота. Оформление — неформальный
бумажный блокнот: тетрадный фон, рукописный шрифт, «скотч», скрепки и слегка
наклонённые карточки. Палитра построена от зелёного `#86a96f` к синим оттенкам.

---

## ✨ Возможности

- **Вход и регистрация** — аккаунт хозяина + паспорт кота, фото профиля.
- **Главная с дашбордом** — сводка: паспорт, ближайшие события, статус прививок.
- **Паспорт кота** — кличка, порода, пол, дата рождения (→ авто-возраст), чип.
- **Породы** — справочник из 15 пород: карточки и модальное окно с деталями
  (история, характер, питание, здоровье, срок жизни). Живой поиск.
- **Медицина и прививки** — записи о здоровье; у прививки можно задать **повтор**
  (раз в год / 3 года) — следующая дата считается автоматически. Фильтр по типу.
- **Календарь** — события по дням с категориями (корм, прививка, прогулка и т.д.),
  показывает привязанные медзаписи и повторы прививок.
- **Трекер веса** — измерения с **графиком динамики** (canvas) и статистикой.
- **Калькулятор корма** — суточная норма по весу, возрасту, активности и
  стерилизации (ветеринарная формула RER → суточная энергия).
- **История кота** — лента постов с текстом и фотографиями.

---

## 🧱 Стек

**Фронтенд** — без фреймворков: HTML + CSS + ванильный JavaScript. Своя
дизайн-система, график веса на `<canvas>`, аккордеоны и модалки без библиотек.

**Бэкенд** — Python + **FastAPI**, **SQLAlchemy 2** + **PostgreSQL**,
авторизация по **JWT** (пароли — bcrypt), загрузка файлов. Контейнеризован Docker.

---

## 🏗️ Архитектура

```
[ Браузер ]
   │  статика (GitHub Pages)
   ▼
Фронтенд (HTML/CSS/JS) ──HTTP/JSON──► Бэкенд FastAPI ──► PostgreSQL
                                          (хост: Render / Docker)
```

Фронтенд — статический сайт, работает сам по себе (хранение в `localStorage`).
Для серверного хранения и синхронизации между устройствами есть полноценный
бэкенд (см. [`backend/`](backend/)) — API под все разделы фронта.

## 🚀 Запуск

### Фронтенд (статика)

Из корня проекта:

```bash
python3 -m http.server 8000
```
Открыть <http://localhost:8000>.

### Бэкенд (API + база) — одной командой

Нужен Docker:

```bash
cd backend
docker compose up --build
```
Поднимется PostgreSQL + API. Документация и тесты API — **<http://localhost:8000/docs>**.
Подробности и запуск без Docker — в [`backend/README.md`](backend/README.md).

## ☁️ Хостинг

- **Фронтенд** → **GitHub Pages** (раздаёт статику).
- **Бэкенд + PostgreSQL** → **Render** (Docker-сервис + база). Конфиг деплоя —
  `render.yaml` в корне (Blueprint). GitHub Pages не запускает Python/БД, поэтому
  бэкенд хостится отдельно.

Пошаговая инструкция по деплою бэкенда — в [`backend/README.md`](backend/README.md).

## 🗄️ Хранение данных

Фронтенд по умолчанию хранит данные в `localStorage` браузера (работает без
сервера). Бэкенд в `backend/` реализует серверное хранение в PostgreSQL: аккаунты,
паспорт кота, медзаписи, события календаря, измерения веса и посты «истории» —
с авторизацией по токену и изоляцией данных по пользователю.
